# Day 1 - Git & GitHub Setup

## 🎯 Goal

Set up the development environment and understand the basics of Git and GitHub.

## ✅ What I Learned

- What Git is
- What GitHub is
- What a Repository is
- Difference between Git and GitHub
- What the `.git` folder does
- What a Branch is
- Why we renamed `master` to `main`
- What the Staging Area is
- What a Commit is
- How Git keeps project history
- How to connect a local repository to GitHub
- How to push commits to GitHub

## 💡 Biggest Learning

Git does not automatically commit every file.
The Staging Area lets me choose exactly what should be included in the next commit.

## 🛠 Commands Learned

- `git init`
- `git status`
- `git add`
- `git commit -m`
- `git log --oneline`
- `git branch -M main`
- `git remote add origin`
- `git push`

## ⚠ Mistake I Made

I created a README both on GitHub and locally, which resulted in unrelated commit histories.

## ✅ How I Solved It

Since the repository was brand new and only I was using it, I safely overwrote the remote repository with my local history.
